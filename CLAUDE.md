# CLAUDE.md — инструкции для агента

Ты помогаешь разрабатывать **simbeam-web** — веб-клиент simbeam (стриминг
iOS-симулятора с Mac, управление из браузера). Это продуктовый репозиторий
клиента; серверная часть — даймон `simbeamd` и брокер `simbeam-signal` —
живёт в основном репо <https://github.com/kei-sidorov/simbeam> и здесь
**не разрабатывается**.

Прежде чем писать код, прочитай:

1. `README.md` — состояние репо и как запустить эталонный клиент.
2. `docs/PROTOCOL.md` — полное описание wire-протокола (акторы, пейринг,
   handshake, control-сообщения, TURN/подписки, глоссарий). Это **снапшот**
   из основного репо; каноника — `docs/HOW-IT-WORKS.md` в simbeam.
3. `reference/client.html` — работающий эталонный клиент одним файлом:
   identity (WebCrypto Ed25519), энролл и реконнект, сигналинг, проверка
   подписи answer'а, видео, ввод (tap/swipe/key), presence. При сомнении
   «как правильно» — сначала смотри сюда.

## Стек и структура

Стек выбран и заскаффолжен (июль 2026):

- **Vite + TypeScript (strict)**, без UI-фреймворка. Рантайм-зависимостей ноль —
  только браузерные API (WebCrypto, WebRTC, WebSocket). Это держит открытие
  страницы быстрым и бандл маленьким (~8 КБ gzip JS).
- **Biome** — линтер и форматтер (`biome.json`).
- **Vitest** — юнит-тесты; чистые части протокола и рендер (jsdom) покрыты.
- **JetBrains Mono** захостен локально (`public/fonts/…woff2`) — никаких
  внешних запросов к Google Fonts.
- **Пейринг только из адресной строки.** Никакого QR-сканера/камеры в вебе:
  пользователь открывает pairing-URL, видит экран подтверждения «Pair this Mac?»,
  жмёт Pair. Фрагмент (`#signal…&daemon…&pair…`) читается из `location.hash`.
- **Деплой** — Cloudflare Pages на `app.simbeam.dev` через GitHub Actions
  (`.github/workflows/ci.yml`, `wrangler.toml`).

Раскладка `src/`:

- `protocol/` — переносимое ядро без DOM: `identity` (Ed25519, storage
  абстрагирован через `KV`), `enroll` (HMAC-пруф + разбор фрагмента),
  `session` (машина сигналинга + RTCPeerConnection + каналы control/bulk),
  `presence` (watch-сокет с бэкоффом), `screenshot` (сборка чанков),
  `messages` (типы wire-протокола). Всё тестируемо в изоляции.
- `app/` — DOM-слой: `store` (наблюдаемое состояние), `controller`
  (вся логика соединения и намерения UI), `render` (три экрана + пейринг),
  `dom`/`icons` (хелперы), `storage` (сохранённые Mac'и в localStorage).
- `main.ts` — точка входа: identity → store → controller → render.

Архитектурное правило: **вся логика соединения живёт в `controller`**, экраны
только наблюдают `store` и отдают намерения (`Intents`). Не тащи вызовы
`Session`/WebRTC в `render`.

## Команды

```bash
npm run dev        # vite dev-сервер
npm test           # vitest (одноразово)
npm run typecheck  # tsc --noEmit
npm run lint       # biome check
npm run build      # typecheck + прод-сборка в dist/
```

Pre-commit хук (`.githooks/pre-commit`, включается `npm install` через
`prepare`) гоняет biome + tsc + vitest. Не коммить в обход хука.

## Правила работы

- **Клиент — только клиент.** Изменения даймона, брокера или протокола —
  в основном репо, не здесь.
- **Протокол не менять в одностороннем порядке.** Расхождение поведения с
  `PROTOCOL.md` — это либо баг клиента, либо устаревший снапшот; сверься с
  канонической версией в simbeam прежде чем «чинить» протокол.
- **Reference-клиент — эталон, не продукт.** Его не развиваем и не
  рефакторим; править можно только если он разошёлся с актуальным протоколом
  (и тогда же обнови снапшот `PROTOCOL.md` с новым коммитом-источником).
- **Проверка — живым подключением.** Медиа-путь и ввод проверяй на реальном
  соединении через локальный стенд (ниже), а не только тестами. Браузер
  не заменяет проверку на целевых устройствах пользователя.
- **Дизайн — минимализм, моноширный.** Токены в `src/style.css`
  (`:root`-переменные): фон `#F1F1F3`, акцент `#373CEE`, зелёный presence
  `#2FBF71`, JetBrains Mono. Держись макета из `Web app QR pairing layout`;
  новые экраны — в той же палитре и на тех же классах (`.card`, `.row`,
  `.btn-*`, `.dot-*`).
- **Никаких новых внешних запросов.** Шрифты, иконки, скрипты — всё локально.
  Прежде чем добавить рантайм-зависимость или CDN-линк, остановись: почти
  наверняка не нужно.
- Вертикальные срезы до запускаемого результата; атомарные коммиты по ходу.

## Локальный стенд

Нужен основной репо simbeam на Mac с Xcode и `idb_companion`:

```bash
make run-remote   # даймон + локальный брокер + отдаёт debug-клиент
```

Нажми **P** в терминале даймона — получишь pairing URL с фрагментом
`#daemon=…&pair=…` (даймон может ещё печатать `signal=…` — клиент его
**игнорирует**, см. ниже). Чтобы проверить страницу из этого репо, запусти
dev-сервер, указав локальный брокер, и открой страницу с этим фрагментом:

```bash
VITE_SIGNAL_URL=ws://localhost:9000/ws npm run dev
# затем открой http://localhost:5173/#daemon=…&pair=…
```

**Брокер — не в URL, а параметр сборки.** Прод всегда ходит на
`signal.simbeam.dev` (`src/config.ts`, `SIGNAL_URL`). Переопределяется только
через env `VITE_SIGNAL_URL` — для локального стенда. Пейринг-URL несёт лишь
`daemon` и `pair`; `parsePairingFragment` не читает `signal`.

Требования браузера: WebCrypto Ed25519 (Chrome 113+/Safari 17+) и secure
context (`localhost` подходит).

Без Mac: `simbeamd demo` (собирается и под linux) стримит headless-Chromium
устройство и печатает **multi-use** pairing URL — им можно энроллить сколько
угодно клиентов.
