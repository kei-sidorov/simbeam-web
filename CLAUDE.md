# CLAUDE.md — инструкции для агента

Ты помогаешь разрабатывать **simbeam-web** — веб-клиент simbeam (стриминг
iOS-симулятора с Mac, управление из браузера). Это продуктовый репозиторий
клиента; серверная часть — даймон `simbeamd` и брокер `simbeam-signal` —
живёт в основном репо <https://github.com/kei-sidorov/simbeam> и здесь
**не разрабатывается**.

Прежде чем писать код, прочитай:

1. `README.md` — состояние репо и как запустить эталонный клиент.
2. `docs/PROTOCOL.md` — полное описание wire-протокола (акторы, пейринг,
   handshake, control-сообщения, TURN/подписки, глоссарий). Это **снапшот**
   из основного репо; каноника — `docs/HOW-IT-WORKS.md` в simbeam.
3. `reference/client.html` — работающий эталонный клиент одним файлом:
   identity (WebCrypto Ed25519), энролл и реконнект, сигналинг, проверка
   подписи answer'а, видео, ввод (tap/swipe/key), presence. При сомнении
   «как правильно» — сначала смотри сюда.

## Правила работы

- **Стек не выбран.** Не скаффолдь фреймворк/сборку/зависимости без явного
  решения пользователя — сначала брейнсторм выбора стека.
- **Клиент — только клиент.** Изменения даймона, брокера или протокола —
  в основном репо, не здесь.
- **Протокол не менять в одностороннем порядке.** Расхождение поведения с
  `PROTOCOL.md` — это либо баг клиента, либо устаревший снапшот; сверься с
  канонической версией в simbeam прежде чем «чинить» протокол.
- **Reference-клиент — эталон, не продукт.** Его не развиваем и не
  рефакторим; править можно только если он разошёлся с актуальным протоколом
  (и тогда же обнови снапшот `PROTOCOL.md` с новым коммитом-источником).
- **Проверка — живым подключением.** Медиа-путь и ввод проверяй на реальном
  соединении через локальный стенд (ниже), а не только тестами. Браузер
  не заменяет проверку на целевых устройствах пользователя.
- Вертикальные срезы до запускаемого результата; атомарные коммиты по ходу.

## Локальный стенд

Нужен основной репо simbeam на Mac с Xcode и `idb_companion`:

```bash
make run-remote   # даймон + локальный брокер + отдаёт debug-клиент
```

Нажми **P** в терминале даймона — получишь pairing URL с фрагментом
`#signal=…&daemon=…&pair=…`. Чтобы проверить страницу из этого репо, подними
статику (`python3 -m http.server`) и открой её, подставив этот фрагмент.

Требования браузера: WebCrypto Ed25519 (Chrome 113+/Safari 17+) и secure
context (`localhost` подходит).

Без Mac: `simbeamd demo` (собирается и под linux) стримит headless-Chromium
устройство и печатает **multi-use** pairing URL — им можно энроллить сколько
угодно клиентов.
